# Windows/Linux 去掉托盘图标，菜单改由悬浮窗右键弹出

悬浮窗已经在任务栏/屏幕上显示了 CPU/内存/上下行四项读数，再放一个系统托盘图标就是重复占位。故 **Windows/Linux 都不再创建托盘图标**，把原本挂在托盘上的菜单（开机启动 / 退出）改为**右键悬浮窗**时用 `WebviewWindow::popup_menu` 在光标处弹出。macOS 不受影响——它的菜单栏托盘图标本身就是显示本体，保留。

## 权衡

- **保留托盘图标**：菜单位置符合系统习惯，但与悬浮窗读数重复，通知区多一个冗余图标。
- **去掉托盘、菜单移到悬浮窗右键（选定）**：通知区更干净；代价是"右键读数出菜单"不是系统惯例，用户需要发现它。

## 后续影响

- 菜单点击事件从 tray 的 `on_menu_event` 上移到 **`Builder::on_menu_event`**，三端共用一套处理。
- 悬浮窗左键只用于拖动（`pointerdown` 仅认 `button===0`），右键专门触发菜单，互不干扰。
- **Windows 风险**：弹出菜单要求宿主窗口是前台窗口才会在点别处时正常 dismiss，而悬浮窗是 `focused(false)` 非激活窗口——若实测菜单点别处不消失，需在 popup 前 `SetForegroundWindow`。
- **Linux 与 Windows 保持功能一致**：无托盘、右键菜单、每秒重新置顶均对齐；`popup_menu` 在各 DE 的表现可能需按桌面环境微调。
