# UI 显示规范 — 两行读数版式

> 悬浮窗（Windows/Linux）与菜单栏托盘图标（macOS）显示的**两行监控读数**的权威版式规范。
> 三端由**同一个 `format_rows()`** 生成同一份字符串，保证逐字符一致。改版式**只改 `format_rows()` 一处**。
> 术语见 [CONTEXT.md](../CONTEXT.md)（「读数」「速度小数点」「悬浮窗」）。

---

## 1. 布局

两行、每行 **9 个等宽字符**，分**左右两组**，中间留 **1 个空格**：

```
idx: 0 1 2 3 4 5 6 7 8
R1:  ↑ . 4 M _ c _ 8 %      →  ↑.4M c 8%    上排：上传速度 + CPU
R2:  ↓ 2 7 K _ m 5 7 %      →  ↓27K m57%    下排：下载速度 + 内存
      └──左组──┘ │ └─右组──┘
                 └ 两组间 1 空格
```

- **左组**（idx 0–3）= `符号 + 右对齐宽 2 的数字 + 单位`，符号 `↑`=上传 / `↓`=下载。
- **右组**（idx 5–8）= `字母 + 右对齐宽 2 的数字 + %`，字母 `c`=CPU / `m`=内存。
- **idx 4** = 两组间的唯一分隔空格。

## 2. 数字格式化规则

- **右对齐到 2 位**：个位数左补 1 个空格（如 CPU 8% → `c 8%`，即 idx6 是空格），好让上下两行的字母列、数字列、`%` 列**竖直对成一条线**。两位数则天然紧凑（`m57%`）。
- **符号/字母紧贴数字**：`↑` 后、`c`/`m` 后**不留空格**（除右对齐补位外）。
- **百分比**：CPU/内存范围 0–100，显示时 `min(99)` 截到 2 位。

### 2.1 速度小数点（关键语义，勿混淆）

网速里的 `.`（如 `.4M`）是**小数点** = **0.4 MB/s**（下一档单位的零点几），用于把跨「字节～吉字节」的速度压到**永远 ≤2 位数字**。

- **它不是"个位数补位符"**。
- **百分比不得使用 `.`**：`.8%` 会被误读成 0.8%（真实是 8%）。CPU/内存单位固定（%）、无换档，没有"零点几档位"。

### 2.2 速度分档表

输入为 **bytes/s**，数字始终 1–2 位，跨档用 `.X` 小数（`.1`–`.9`，不出现 `.0`）：

| 速度范围 | 格式 | 示例 |
|----------|------|------|
| 0–99 B/s | `XXB` | `7B`、`63B` |
| 100–1023 B/s | `.XK` | 512B/s → `.5K` |
| 1–99 KB/s | `XXK` | `27K` |
| 100–1023 KB/s | `.XM` | 410KB/s → `.4M` |
| 1–99 MB/s | `XXM` | `55M` |
| 100–1023 MB/s | `.XG` | 500MB/s → `.4G` |
| ≥ 1 GB/s | `XXG` | `1G`（`min(99)`） |

小数计算：`.{ (bps / 下一档 * 10).floor().clamp(1, 9) }`。

## 3. 三端渲染

| 平台 | 载体 | 字体 | 尺寸 | 深/浅色 |
|------|------|------|------|---------|
| **macOS** | 菜单栏托盘图标（`render_icon` 光栅化进 `CGBitmapContext`） | Core Text **加粗**系统字体（`kCTFontEmphasizedSystemFontType`），逐字符居中在 14px 等宽格子、字号 18px(@2x) | `ICON_W=66` 逻辑像素（9 列 × 14px + 内边距） | 黑字 + `isTemplate` 系统自动反相 |
| **Windows / Linux** | 悬浮 WebView 小窗（`ui/index.html`） | 等宽 `ui-monospace/Cascadia/Consolas` 700 粗、`white-space:pre` 保留对齐空格 | 窗口 `inner_size` 82×36；淡底药丸 `fit-content` | 白字 + `text-shadow` 提对比 |

- **等宽是版式对齐的前提**：三端都用等宽字体/格子，`format_rows()` 里的空格才能逐字符对上。
- **空格 = 一个字符格**：`c 8%` 里那个对齐空格在 macOS 占一个 14px 空格子、在 Web 端占一个 `pre` 空格，视觉上和一个字符等宽——这是刻意的，为了列对齐。

## 4. 改版式的唯一入口

**只改 `format_rows()`**（`src-tauri/src/lib.rs`），三端同步。改完：

- macOS 用 `cargo test --lib preview_icon` 导出 PNG 肉眼核对；
- 若列数变化，同步调 `ICON_W`（macOS）与 `inner_size`（Win/Linux）容纳新宽度；
- 同步更新本文件、`MetricsPayload` 示例注释、`ui/index.html` 占位文本。
